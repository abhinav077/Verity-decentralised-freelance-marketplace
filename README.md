# Verity — Decentralized Freelancing Marketplace

> A fully on-chain, trustless freelancing platform powered by Ethereum/Polygon smart contracts, a soulbound reputation system, and a DAO governance layer.

---

## Overview

Verity DFM is an open, permissionless freelancing marketplace where clients and freelancers interact directly through smart contracts — no intermediaries, no hidden fees, and no central authority. Payments are held in escrow, disputes are resolved by a decentralized jury, and reputation is built on-chain through a non-transferable token system.

The platform is currently **fee-free** (0% platform fee) to encourage early adoption. A DAO treasury exists for community-governed proposals and crowdfunding.

---

## Features

### Core Marketplace
- **Job Market** — Post jobs, submit bids (with optional milestone breakdowns), negotiate, and settle — all on-chain.
- **Escrow** — Funds are locked in a smart contract at job creation and released only when work is approved, a dispute is resolved, or a mutual settlement is reached. Auto-release triggers after 14 days of inactivity.
- **Milestones** — Clients can structure payments across multiple deliverable checkpoints.
- **Mutual Cancellation / Settlement** — Either party can propose a split; both must agree before funds move.

### Reputation System
- **VRT (Verity Reputation Token)** — Soulbound (non-transferable) ERC-20 token earned by completing jobs and contributing to the platform. It can never be bought, only earned.
- **Tiers** — Bronze → Silver (50 VRT) → Gold (200 VRT) → Platinum (500 VRT), unlocking governance weight and jury eligibility.

### User Profiles
- On-chain profiles with name, bio, profile photo (IPFS), and skills.
- Bidirectional reviews (client ↔ freelancer) locked to completed jobs.
- Skill endorsements between users who have worked together.
- Portfolio entries — supports IPFS file uploads AND regular links (GitHub, website, etc.).
- Admin-verified skill badges.

### Dispute Resolution
- Disputed jobs go to a jury of VRT-holding peers, weighted by tier.
- 50-50 ties trigger a re-vote instead of auto-resolving.
- Disputes auto-resolve after 10 days; initiator wins if no votes are cast.
- Disputes can be escalated to admin arbitration after a 7-day grace period.

### Bounty Board
- Anyone can post an open bounty with an ETH prize.
- Multiple submissions; poster approves winners.
- Silver-tier or higher required to post.

### Sub-Contracting
- Accepted freelancers can bring in sub-contractors and split payment at the smart-contract level.

### DAO Governance & Treasury
- VRT-weighted on-chain proposals and voting.
- Governance treasury for community funds.
- Crowdfunding module for community-driven public-good projects (creators can withdraw funds as they are raised).

## Getting Started — Full Step-by-Step Installation

> This guide is written for first-timers. Follow every step in order.

### Step 1: Install prerequisites

You need these programs installed on your computer before anything else:

| Program | Why you need it | How to install |
|---|---|---|
| **Node.js** (v18 or newer) | Runs all the JavaScript/TypeScript code | https://nodejs.org — download the LTS version and run the installer |
| **npm** | Installs packages (comes with Node.js) | Automatically installed with Node.js |
| **Git** | Clones the project from GitHub | https://git-scm.com/downloads |
| **MetaMask** | Crypto wallet for your browser | https://metamask.io — install the browser extension |

**Verify Node.js is installed** — open a terminal and run:
```bash
node --version
# Should print v18.x.x or higher
```

### Step 2: Clone the repository

```bash
git clone https://github.com/abhinav077/Verity-decentralised-freelance-marketplace.git
cd DFM
```

### Step 3: Install dependencies

You need to install packages for **both** the contracts and frontend folders.

```bash
# Install contract dependencies
cd contracts
npm install

# Go back up and install frontend dependencies
cd ../frontend
npm install
```

> This may take a few minutes. Wait for both to finish.

### Step 4: Set up environment files

You need to create two config files that tell the app your private info (wallet key, API keys). These files are **never pushed to GitHub** — they stay on your machine only.

#### 4a. Contracts environment file

```bash
cd ../contracts
cp .env.example .env
```

Now open `contracts/.env` in a text editor and fill in your **private key**:

```dotenv
# ⚠️ NEVER share your private key or commit it to GitHub!
PRIVATE_KEY=your_wallet_private_key_here

# This is already set — no changes needed for Polygon Amoy:
POLYGON_AMOY_RPC_URL=https://rpc-amoy.polygon.technology
```

**How to get your private key from MetaMask:**
1. Open MetaMask → click the ⋮ menu next to your account
2. Click **Account details**
3. Click **Show private key** → enter your password
4. Copy the key (it starts with a long hex string like `62a85c...`)
5. Paste it into the `PRIVATE_KEY=` line

> **Security tip:** Use a **dedicated testnet-only wallet**. Never use a wallet that holds real money.

#### 4b. Frontend environment file

```bash
cd ../frontend
cp .env.example .env.local
```

Now open `frontend/.env.local`. The contract addresses will be auto-filled when you deploy (Step 6). For now, only the **Pinata JWT** needs to be added manually:

```dotenv
# The deploy script will auto-fill these:
NEXT_PUBLIC_CHAIN_ID=80002
NEXT_PUBLIC_RPC_URL=https://rpc-amoy.polygon.technology
NEXT_PUBLIC_VRT_TOKEN=
NEXT_PUBLIC_JOB_MARKET=
# ... etc (leave blank for now)

# YOU need to fill this in (see Step 5):
PINATA_JWT=your_pinata_jwt_here
NEXT_PUBLIC_PINATA_GATEWAY=https://gateway.pinata.cloud/ipfs
```

### Step 5: Get a Pinata JWT token (for IPFS file uploads)

Pinata lets you upload files (profile photos, portfolio items, dispute evidence) to IPFS. It's free.

1. Go to https://app.pinata.cloud and **create a free account**
2. After signing in, click **API Keys** in the left sidebar
3. Click **+ New Key**
4. Under permissions, enable:
   - ✅ `pinFileToIPFS`
   - ✅ `pinJSONToIPFS`
5. Give it a name like "Verity DFM" and click **Create Key**
6. You'll see three values — copy the **JWT** (the longest one)
7. Paste it into `frontend/.env.local`:
   ```
   PINATA_JWT=eyJhbGciOiJIUzI1N...your_jwt_here
   ```

> The JWT is a single long string starting with `eyJ`. Copy the whole thing.

### Step 6: Get testnet tokens and deploy contracts

#### 6a. Add Polygon Amoy to MetaMask

1. Open MetaMask → click the network dropdown at the top
2. Click **Add a custom network** (or Networks → Add Network)
3. Fill in:
   - **Network Name:** `Polygon Amoy`
   - **RPC URL:** `https://rpc-amoy.polygon.technology`
   - **Chain ID:** `80002`
   - **Currency Symbol:** `POL`
   - **Block Explorer:** `https://amoy.polygonscan.com`
4. Click **Save** and switch to Polygon Amoy

#### 6b. Get free testnet POL tokens

You need (fake) POL tokens to pay for gas fees on the testnet:

1. Go to https://faucet.polygon.technology/
2. Select **Amoy** network
3. Paste your MetaMask wallet address
4. Click **Submit** — you'll receive free testnet POL in a few minutes

> Check your MetaMask balance. You should see some POL appear. If not, wait a minute and refresh.

#### 6c. Deploy the smart contracts

```bash
cd ../contracts
npm run deploy:amoy
```

This will:
- Compile all 10 contracts
- Deploy them to the Polygon Amoy testnet
- **Automatically create/update** `frontend/.env.local` with all contract addresses
- Preserve your existing Pinata JWT in the file

You should see output like:
```
Deploying DFM contracts...
VRTToken deployed to: 0xEf9A...
JobMarket deployed to: 0x8EF7...
...
✅ Deploy complete! Addresses written to ../frontend/.env.local
```

> **If it fails**: Make sure your wallet has testnet POL and your private key is correct in `contracts/.env`.

### Step 7: Start the frontend

```bash
cd ../frontend
npm run dev
```

Open http://localhost:3000 in your browser. That's it!

> **Important:** After deploying to a testnet, your data is **permanent** on the blockchain.
> You only need `npm run dev` to start the frontend. No need to redeploy unless you change the smart contracts.

## Day-to-Day Usage

Once you've done the initial setup above, here's what you need each time:

```bash
# Just start the frontend — contracts are already deployed on-chain
cd frontend
npm run dev
```

Open http://localhost:3000, connect your MetaMask wallet (make sure you're on Polygon Amoy), and you're ready to go.

**When do you need to redeploy?**
- Only if you modify the Solidity smart contracts themselves
- Changing the frontend code does NOT require redeployment — just restart the dev server

## Alternative: Local Development (for testing without testnet)

If you just want to test quickly without needing testnet tokens:

```bash
# Terminal 1 — start a local blockchain
cd contracts
npm run node

# Terminal 2 — deploy to the local chain
cd contracts
npm run deploy:local

# Terminal 3 — start the frontend
cd frontend
npm run dev
```

MetaMask local config:
- **RPC URL:** `http://127.0.0.1:8545`
- **Chain ID:** `31337`
- Import a test private key from the terminal output (Hardhat gives you 10 pre-funded accounts)

> **Warning:** Local chain data resets when you stop the node (unless you use the start-node.sh script with Anvil state persistence).

## Troubleshooting

| Problem | Solution |
|---|---|
| "Wrong Network" badge in navbar | Click the badge — it will prompt MetaMask to switch to the correct network automatically |
| "Contracts not found" on profile page | Make sure you deployed contracts and `frontend/.env.local` has addresses filled in. Restart the dev server after deploying. |
| MetaMask won't connect | Make sure MetaMask is installed and unlocked. Refresh the page. |
| IPFS uploads fail | Check that `PINATA_JWT` is set correctly in `frontend/.env.local`. The JWT is server-side only (no `NEXT_PUBLIC_` prefix). |
| Deploy fails with "insufficient funds" | Get more testnet tokens from the faucet. |
| Deploy fails with "nonce too high" | Reset your MetaMask account: Settings → Advanced → Clear activity tab data |
| `npm install` fails | Delete `node_modules` and `package-lock.json`, then run `npm install` again |
